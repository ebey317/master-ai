Why this page is blocked

The extension classifies hosts into four categories using a local list at ~/.master_ai_domain_classes.json. Two of those categories cause hard blocks:

  • Category 1 — known malicious / phishing. The host appears on a list of credential-harvest, malware-drop, or known fraud sites. Sensei will not navigate, click, fill, or read this page.
  • Category 2 — sensitive auth surface. Banking, healthcare, government portals, and similar surfaces where automated action carries unrecoverable risk. Use these sites yourself in a regular tab — Sensei stays out.

Category 3 hosts (adult / gambling / crypto-exchange) are not blocked, but every action there requires a per-action confirm.

If you believe this is wrong

The list is local and editable. To inspect the matched entry, open ~/.master_ai_domain_classes.json. The category buckets are category_1, category_2, and category_3; each entry is "domain.example": "reason text". Removing an entry takes effect on the next classifier call (cached for 60 seconds).

The list refreshes weekly via the maintenance window (~/scripts/refresh_domain_classes.sh); manual edits survive until the next refresh.
